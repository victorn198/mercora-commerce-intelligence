# Project Instructions

- Keep every dependency, cache, temporary file, and dataset inside this project on `D:`.
- Use `run.cmd` for Python commands and `tool.cmd` for executable tools.
- Never commit raw Olist CSV or zip files.
- Never change a metric definition without updating `docs/METRIC_DICTIONARY.md` and tests.
- Keep order, item, and payment facts separate until each is aggregated to order grain.
- Do not claim current or live data. The source period is 2016-2018.
- Do not create commits unless the user explicitly requests one.
